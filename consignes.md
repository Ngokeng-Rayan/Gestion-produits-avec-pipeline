**MISE EN PLACE D’UNE CHAÎNE CI/CD COMPLÈTE**

**Contexte**

Dans un environnement de développement moderne, les entreprises doivent livrer r
apidement des applications fiables et performantes. L’approche **DevOps** permet
 d’automatiser les processus de développement, de test, de déploiement et de sup
ervision.

Vous êtes une équipe DevOps chargée de mettre en place une chaîne **CI/CD complè
te**, incluant la mise en production, le monitoring et l’évolution de l’infrastr
ucture.

**Objectifs pédagogiques**

À l’issue de ce projet, vous serez capables de :

- Mettre en place une pipeline **CI/CD automatisée**
- Conteneuriser une application avec Docker
- Superviser une application avec Prometheus et Grafana
- Mettre en œuvre des stratégies de déploiement avancées
- Implémenter des bonnes pratiques DevOps (tests, sécurité, automatisation)

**Description du projet**

Vous devez concevoir, implémenter et déployer une application (au choix) avec un
e chaîne CI/CD complète.

**Exemples d’applications possibles**

- Application web de gestion de recettes
- API REST (gestion d’étudiants, produits, etc.)
- Plateforme e-learning simplifiée
- Application mobile connectée à une API

**PARTIE 1 : MISE EN PLACE DE LA CI/CD**

**Travail demandé**

1. Créer un repository Git (GitHub ou GitLab)
2. Travail en équipe avec 4 collaborateurs minimum
3. Mettre en place une pipeline CI avec GitHub Actions ou GitLab CI/CD :
  - Build automatique
  - Linting (qualité du code)
  - Tests unitaires
4. Conteneuriser l’application avec Docker :
  - Écriture d’un Dockerfile
  - Configuration de docker-compose
5. Mettre en place un pipeline CD :
  - Déploiement automatique sur un serveur (staging)
  - Déploiement manuel ou automatique en production

**Livrables**

- Code source sur Git
- Fichiers de configuration CI/CD
- Image Docker fonctionnelle
- URL de l’application déployée

**PARTIE 2 : MONITORING ET LOGS**

**Travail demandé**

1. Mettre en place un système de monitoring avec :
  - Prometheus (collecte de métriques)
  - Grafana (visualisation)
2. Superviser :
  - CPU, RAM, réseau
  - Temps de réponse de l’application
  - Taux d’erreur
3. Mettre en place la gestion des logs avec :
  - ELK Stack
4. Configurer un système d’alertes :
  - Email ou notification en cas de panne

**Livrables**

- Dashboard Grafana
- Captures d’écran des métriques
- Description des alertes configurées

**PARTIE 3 : ÉVOLUTION ET OPTIMISATION**

**Travail demandé**

1. Déployer l’application avec Kubernetes :
  - Pods, services, déploiements
2. Mettre en place :
  - Auto-scaling
  - Rolling update ou Blue/Green deployment
3. Ajouter :
  - Tests de performance
  - Scan de sécurité avec SonarQube
4. Automatiser l’infrastructure avec :
  - Terraform (optionnel bonus)

**Livrables**

- Fichiers Kubernetes (YAML)
- Pipeline CI/CD amélioré
- Rapport technique détaillé

**Livrables finaux**

Chaque étudiant devra fournir :

- Code source (Git)
- Rapport (10–15 pages) :
  - Architecture du système
  - Description de la pipeline CI/CD
  - Monitoring et résultats
  - Difficultés rencontrées
- Démonstration (5–10 minutes)

**Organisation**

- Travail **individuel**
- Durée : **3 à 4 semaines**

**BONUS**

- Mise en place de DevSecOps
- Dashboard métier (KPI)
- Déploiement multi-environnement (dev/staging/prod)
- Intégration frontend + backend complète

**Contraintes**

- Utilisation obligatoire de Docker
- Pipeline CI/CD automatisée obligatoire
- Monitoring obligatoire
- Respect des bonnes pratiques (code, documentation)

**Conseils**

- Commencez simple (CI/CD basique), puis itérez
- Testez chaque étape avant de passer à la suivante
- Documentez tout (très important !)

**Résultat attendu**

Une application **entièrement automatisée, déployée, monitorée et scalable**, co
nforme aux standards DevOps actuels.
